# Ralphton 해커톤 회고 (비개발자용)

## 이 문서는?

21시간 동안 진행된 AI 해커톤에서 무엇을 시도했고, 어디서 막혔고, 무엇을 배웠는지 정리한 글입니다. 기술 용어를 최대한 풀어서 설명합니다.

---

## 우리가 만들려던 것

**레고 블록을 옮기는 AI 로봇 조종사**를 만드는 것이 목표였습니다.

사람이 직접 로봇을 조종하는 영상을 AI에게 보여주고, AI가 그 영상을 보고 "아, 이런 상황에서는 이렇게 움직이면 되는구나"를 스스로 배우게 하는 방식입니다. 마치 운전교습소에서 교관의 시범을 보고 배우는 것과 비슷합니다.

### 구체적 과정

1. **시범 영상 만들기** — 사람(전문가)이 로봇을 조종하는 영상을 대량으로 만든다
2. **AI 학습시키기** — 그 영상들을 AI에게 보여주며 패턴을 익히게 한다
3. **실전 테스트** — 학습한 AI가 직접 로봇을 조종하게 해본다
4. **피드백 반영** — 실전에서 실패하면 원인을 분석하고 시범 영상을 보완한다
5. **1~4를 반복** — 이 사이클을 10회 돌려 점점 나아지게 한다

---

## 목표 vs 실제 결과

- **목표**: 위 사이클을 10회 반복
- **실제**: 3회 반복 + 추가 보완 작업 (미완료 상태로 종료)
- **AI의 실전 성공률**: 0% (시범은 잘 따라했지만, 혼자 하면 실패)

---

## 왜 목표를 달성하지 못했나? (3대 병목)

### 병목 1: 시범 영상을 만드는 데 시간이 너무 오래 걸렸다

목표는 시범 영상 1,000개였지만, 실제로는 286개만 만들었습니다 (28.6%).

- 영상 1개 만드는 데 **4~8분** 소요
- 1,000개면 약 **100시간** 필요 → 21시간 해커톤에서 물리적으로 불가능
- 나중에 해상도를 낮춰 2~3분으로 줄였지만, 이미 시간이 부족

**핵심 실수**: 영상을 한 번에 1개씩만 만들었습니다. 처음부터 여러 개를 동시에 만들었다면 (컴퓨터 여러 대를 쓰는 것처럼) 시간을 1/4~1/8로 줄일 수 있었습니다.

### 병목 2: 시험에서는 만점, 실전에서는 0점 문제

AI의 학습 성적표는 훌륭했습니다. 시험 점수로 치면 거의 만점이었습니다. 그런데 실전에서는 **한 번도 성공하지 못했습니다.**

왜 이런 일이 벌어졌을까요?

- 시범 영상이 **"잘되는 경우"만** 보여줬습니다
- 실전에서 조금이라도 어긋나면 AI가 **"이런 상황은 배운 적 없는데?"** 상태가 됩니다
- 어긋난 상태에서 복구하는 방법을 배운 적이 없으니, 계속 잘못된 방향으로 갑니다

**비유하면**: 운전을 배울 때 직진과 좌회전만 연습한 학생이 갑자기 유턴해야 하는 상황을 만나면 당황하는 것과 같습니다. "실수했을 때 어떻게 복구하는지"도 같이 가르쳤어야 했습니다.

나중에 "실수 복구" 시범도 추가했지만, 이번에는 "벽에 부딪혀서 꼼짝 못하는" 새로운 문제가 생겼습니다.

### 병목 3: AI 학습 자체에 시간이 오래 걸렸다

- 첫 번째 학습: **5시간 13분** (밤 11시 시작 → 새벽 4시 반 완료)
- 이후 보완 학습: 1~1.5시간씩

첫 학습이 끝날 때까지 5시간을 그냥 기다렸는데, 이 시간에 다음 버전의 시범 영상을 미리 만들었어야 했습니다.

---

## 21시간이 실제로 어떻게 흘러갔나

- **저녁 8시~11시** — 시뮬레이터 세팅, 시범 영상 52개 제작
- **밤 11시~새벽 4시반** — AI 학습 중 (5시간 대기)
- **새벽 3시~4시** — 학습 결과 확인 → 실전 성공률 0% 발견
- **새벽 4시~6시** — 원인 분석, 시범 영상 200개 추가 제작
- **새벽 6시~7시** — 보완 학습 → 여전히 0%
- **아침 7시~9시** — 벽 충돌 문제 발견, 추가 보완 시도 → 시간 종료

**가장 아까운 구간**: 밤 11시~새벽 4시반, 학습을 기다리는 5시간이 완전한 공백이었습니다.

---

## 인프라 문제들

해커톤 중에 기술적 트러블도 있었습니다.

- **클라우드 인증 만료**: 새벽 4시쯤 클라우드 서버 접속 권한이 만료되어, AI 에이전트들의 상태를 자동으로 확인하던 감시 시스템이 작동을 멈췄습니다
- **파일 전송 실패**: 서버 간 파일 전송이 안 되어 우회 경로를 찾느라 시간 소모
- **에이전트 무응답**: AI 에이전트 중 하나가 33분간 응답하지 않아, 감시 시스템이 4번 재시동을 시도한 끝에 복구

---

## 잘된 점

- **5개의 AI 에이전트가 13시간 넘게 자율적으로 협업**한 것 자체는 큰 성과
- 감시 시스템이 문제를 5분 이내에 감지하고 복구
- AI 학습 자체는 매우 잘 수렴 (시험 성적은 거의 만점)
- 실전 실패 원인을 해커톤 안에서 정확히 진단해냈음
- 시간이 부족했지만 보완 전략을 설계하고 부분적으로 검증까지 진행

---

## 핵심 교훈 3가지

### 1. 시험 점수가 높다고 실전에서 통하는 게 아니다

AI 학습에서 "오프라인 성적"이 아무리 좋아도 실제 환경에서 동작하는지는 별개입니다. **실전 테스트를 처음부터 빨리 자주 해봤어야** 했습니다. 5시간 학습 후 "0% 성공"을 발견하면 이미 너무 늦습니다.

### 2. 속도를 먼저 측정하라

"시범 영상 1,000개"라는 목표는 **"영상 1개 만드는 데 몇 분 걸리는지"를 한 번도 측정하지 않고** 세운 숫자였습니다. 4~8분이라는 걸 알았다면 처음부터 동시에 여러 개를 만들거나, 목표를 현실적으로 조정했을 것입니다.

### 3. 기다리는 시간을 낭비하지 마라

AI가 5시간 동안 학습하는 걸 기다린 시간이 가장 큰 낭비였습니다. 학습이 돌아가는 동안 **다음 단계를 미리 준비**했어야 했습니다. 한 작업이 돌아가는 동안 다른 작업을 병렬로 진행하는 것이 중요합니다.

## 사용된 비용

### 클라우드 서버 (GCP VM 5대)

5개의 AI 에이전트가 각각 별도 서버에서 돌아갔습니다.

**서울 리전 (4대)** — 에이전트 간 통신이 빨라야 하므로 가까운 서버 사용

- **Watcher (감시자)**: 8코어/32GB 메모리 — 전체 진행 관리, 에이전트 상태 감시
- **Developer (개발자)**: 8코어/32GB 메모리 — 시뮬레이터 개발, 시범 영상 생성
- **DomainExpert (전문가)**: 4코어/16GB 메모리 — 학습 시나리오 설계
- **Evaluation (평가자)**: 4코어/16GB 메모리 — AI 성능 검증, 실패 분석

**미국 리전 (1대)** — AI 학습 전용 GPU 서버가 미국에만 있어서 별도 배치

- **Training (학습)**: A100 GPU 40GB — AI 모델 학습 전용 (Spot 인스턴스로 비용 절감)

### 예상 비용 내역

**클라우드 서버 비용**

- 일반 서버 4대 (서울): 약 $6 (시간당 $0.15~0.30 × 가동시간)
- GPU 서버 1대 (미국 A100): 약 $14 (시간당 ~$2.48 × 약 5.5시간)
  - Spot 인스턴스 사용 — 유휴 GPU를 싸게 빌리는 방식으로 정가 대비 60~70% 할인

**AI API 호출 비용**

- Claude Opus 4.6 (Developer 에이전트): API 토큰 사용
- GPT-5.2 Pro (DomainExpert 에이전트): API 토큰 사용
- 예상 API 비용: 약 $50~100

**클라우드 저장소 (GCS)**

- 학습 데이터, 체크포인트 등 약 20GB 전송: 약 $1 미만

**Discord 봇**

- 5개 봇 인스턴스, 13시간+ 가동, 200+ 메시지 교환: 무료

### 총 예상 비용: 약 $70~120 (약 9만~16만원)

### 비용 관련 참고사항

- **GPU가 놀았다**: A100 40GB 중 실제 사용은 3.4GB (8.5%). 더 작고 저렴한 GPU로도 충분했을 가능성이 높음
- **Spot 인스턴스 활용**: GPU 서버를 정가가 아닌 할인가로 빌려 비용 절감
- **가장 비싼 항목은 AI API 호출**: 서버보다 에이전트들이 대화하면서 소모한 API 비용이 더 큼
- **별도 비용 추적 없었음**: 해커톤 특성상 사전 예산 관리 없이 진행

## 용어 정리

- **에피소드**: AI에게 보여주는 시범 영상 1편. "이런 상황에서는 이렇게 하면 돼"를 보여주는 교육 자료
- **학습 (Training)**: AI가 시범 영상들을 반복적으로 분석하며 패턴을 익히는 과정
- **Rollout**: 학습한 AI를 실전 환경에 놓고 혼자 해보게 하는 테스트
- **Epoch**: AI가 전체 시범 영상을 한 바퀴 학습하는 단위. 500 epoch = 전체 영상을 500번 반복 학습
- **Loss**: AI의 실수 크기를 숫자로 나타낸 것. 낮을수록 좋음 (0에 가까울수록 시범과 비슷하게 행동)
- **Covariate Shift**: "배운 상황"과 "실전 상황"이 달라서 AI가 헤매는 현상
- **ACT**: 이번에 사용한 AI 학습 방법의 이름 (Action Chunking with Transformers)
- **headless 렌더링**: 화면에 보여주지 않고 컴퓨터 내부에서만 시뮬레이션을 돌리는 방식
- **A100**: 이번에 사용한 고성능 GPU (AI 학습 전용 하드웨어)
